# 개발 목적
## 1. 기술적 도전: 지능형 AI 에이전트 아키텍처 구현

가장 큰 목적은 LLM이 단순히 텍스트 응답만 하는 것을 넘어, **스스로 판단하고 행동(Tool Calling)하는 시스템**을 구축하는 능력을 배양하는 것입니다.

* **Multi-modal 처리:** 이미지 데이터를 텍스트로 전환(Vision)하고, 이를 다시 수치 데이터(Embedding)로 바꾸는 데이터 파이프라인을 경험합니다.
* **지능형 필터링:** 단순 검색이 아닌, 유저의 과거 취향(Vector DB)과 현재 요청 카테고리를 결합한 '컨텍스트 기반 검색'을 구현합니다.
* **에이전트 제어:** LangChain을 이용해 LLM이 언제 외부 검색 API를 호출해야 할지 스스로 판단하게 만드는 로직을 학습합니다.

## 2. 엔지니어링 역량: 엔터프라이즈급 풀스택 환경 숙련

학습용 프로젝트이지만, 실제 서비스 환경과 동일한 인프라를 직접 구성해 보는 것이 목적입니다.

* **구조화된 백엔드:** NestJS의 모듈형 아키텍처를 통해 AI 로직과 비즈니스 로직을 깔끔하게 분리하는 설계 감각을 익힙니다.
* **컨테이너 기반 개발:** Docker와 Docker Compose를 활용해 프론트엔드, 백엔드, 벡터 DB가 유기적으로 돌아가는 MSA(Microservices Architecture)의 기초를 다집니다.
* **메타데이터 관리:** 벡터 데이터에 메타데이터(카테고리 등)를 부여하고 필터링하는 실무형 벡터 DB 활용 기술을 습득합니다.

## 3. 사용자 가치: 개인화된 시각적 경험의 자동화

사용자 입장에서는 자신의 취향을 굳이 말로 다 설명하지 않아도, 몇 장의 사진만으로 AI가 나의 페르소나를 이해하게 만드는 '개인화 경험'을 제공하는 것이 목적입니다.

* **추상적 취향의 구체화:** "느낌 있는 사진" 같은 모호한 요청을 AI가 유저의 과거 데이터를 바탕으로 "어두운 톤의 미니멀한 건축 사진"처럼 구체적인 쿼리로 변환해 줍니다.
* **카테고리별 전문 비서:** 인물, 풍경 등 목적에 맞는 취향 서랍을 따로 관리하여 상황에 맞는 최적의 결과물을 제공합니다.
