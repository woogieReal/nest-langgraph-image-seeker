# 개발 계획
## 🧰 최종 확정된 풀스택 기술 스택

| 영역 | 기술 스택 | 주요 역할 |
| --- | --- | --- |
| **Frontend** | React (Vite), Tailwind CSS, shadcn/ui | 사용자 인터페이스 구성 및 API 통신 (`useState`로 상태 관리) |
| **Backend** | NestJS (TypeScript) | API 서버, AI 에이전트 로직, 파일 처리 |
| **AI & Agent** | LangChain, OpenAI API (Vision, Embedding) | 이미지 분석(태깅), 메타데이터 기반 RAG 검색, 외부 API 호출 |
| **Database** | PostgreSQL + pgvector (TypeORM) | 유저 정보, 카테고리(메타데이터), 임베딩 벡터 저장 |
| **Infrastructure** | Docker, Docker Compose | 프론트엔드, 백엔드, DB를 하나의 네트워크로 묶어 컨테이너화 |

---

## 🗓️ 5단계 풀스택 상세 개발 마일스톤

### **1단계: 도커 기반 통합 개발 환경 구축 (Infra)**

프론트엔드와 백엔드 레포지토리를 구성하고 도커로 하나로 묶습니다.

* **목표:** React, NestJS, pgvector 컨테이너 동시 실행
* **세부 작업:**
* Vite를 사용하여 React 프로젝트 생성 및 Tailwind CSS, shadcn/ui 초기 세팅.
* NestJS 프로젝트 생성 및 TypeORM 엔티티(카테고리, 벡터 컬럼 포함) 설정.
* 루트 디렉토리에 `docker-compose.yml`을 작성하여 `frontend(포트 5173)`, `api(포트 3000)`, `db(포트 5432)` 서비스 구성.
* 컨테이너 간 네트워크 통신 테스트 (React -> NestJS API 호출 확인).



### **2단계: 프론트엔드 UI 뼈대 및 상태 설계 (Frontend UI)**

가벼운 상태 관리(`useState`)를 활용해 직관적인 대시보드 화면을 만듭니다.

* **목표:** shadcn/ui를 활용한 메인 레이아웃 완성
* **세부 작업:**
* **사이드바 (기억 주입구):** 카테고리 선택 드롭다운(`Select` 컴포넌트)과 이미지 업로드 폼 구성. 업로드 대기 중인 파일은 `useState`로 관리.
* **메인 채팅창:** 에이전트와 대화할 수 있는 채팅 인터페이스(`ScrollArea`, `Input`, `Button`) 구현. 대화 내역 배열을 `useState`로 업데이트.
* **결과 갤러리:** 에이전트가 찾아온 이미지들을 보여줄 그리드 레이아웃(Card 기반) 구성.



### **3단계: 백엔드 '기억' 파이프라인 및 프론트 연동 (Memory Pipeline)**

사용자가 올린 사진을 AI가 분석하고 저장하는 핵심 기능을 연결합니다.

* **목표:** 이미지 업로드 -> Vision 분석 -> 벡터 DB 저장 흐름 완성
* **세부 작업:**
* **NestJS:** `POST /preferences/upload` API 구현. 이미지를 받아 GPT-4o(Vision)로 분위기 키워드를 추출하고, 임베딩을 거쳐 카테고리와 함께 DB에 저장.
* **React:** 사이드바에서 이미지를 선택하고 '학습하기'를 누르면 해당 API로 `FormData`(이미지+카테고리)를 전송하고, 성공 시 알림(`Toast`) 띄우기.



### **4단계: 에이전트 검색 파이프라인 및 채팅 연동 (Agent Pipeline)**

채팅창에서 특정 카테고리를 요구하면 에이전트가 검색을 수행하게 만듭니다.

* **목표:** 카테고리 필터링 기반 RAG 에이전트 완성
* **세부 작업:**
* **NestJS:** `POST /agent/chat` API 구현. LangChain Retriever에 메타데이터 필터(`category`)를 적용하고, 외부 이미지 검색 API를 호출하는 Tool을 Agent에 장착.
* **React:** 채팅창에 "풍경 사진 찾아줘"라고 입력 시 API를 호출하고, 로딩 상태를 보여주다가 응답이 오면 대화창에 텍스트를, 갤러리 영역에 이미지 결과물을 렌더링.



### **5단계: 디테일 최적화 및 에러 핸들링 (Polish)**

서비스의 완성도를 높이는 마지막 단계입니다.

* **목표:** 사용자 경험(UX) 개선 및 안정성 확보
* **세부 작업:**
* 외부 이미지 링크가 깨졌을 때를 대비한 프론트엔드 이미지 Fallback 처리.
* API 호출 중일 때 버튼 비활성화 및 스피너 UI 추가.
* 도커 볼륨 설정 재확인(컨테이너 재시작 시 DB 데이터 유지 여부 점검).
